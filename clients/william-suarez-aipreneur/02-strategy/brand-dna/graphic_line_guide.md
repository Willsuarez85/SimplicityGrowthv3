## 5. Lenguaje Grafico (CRITICO)

### 5.1 Diagrama: INPUT → PROCESS → OUTPUT

El patron mas basico. Usalo para explicar cualquier transformacion.

```
┌──────────┐      ┌──────────────┐      ┌──────────┐
│  INPUT   │ ───► │   PROCESS    │ ───► │  OUTPUT  │
│          │      │              │      │          │
│ [datos]  │      │ [accion/AI]  │      │[resultado]│
└──────────┘      └──────────────┘      └──────────┘
                         │
                    ┌────┴────┐
                    │ (verde) │
                    │ punto   │
                    │ clave   │
                    └─────────┘
```

**Reglas:**
- Maximo 3-5 bloques en horizontal
- Flechas simples, no decoradas
- Verde SOLO en el elemento mas importante
- Etiquetas cortas (2-3 palabras max)

---

### 5.2 Diagrama: Human in the Loop (OBLIGATORIO)

**El humano SIEMPRE decide.** Este diagrama es el core visual de AIpreneurs.

```
    ┌─────────┐
    │ TRIGGER │
    │ (input) │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │   AI    │
    │  AGENT  │
    │ (draft) │
    └────┬────┘
         │
         ▼
┌─────────────────┐
│  HUMAN REVIEW   │ ◄── Verde aqui: EL HUMANO DECIDE
│   (decision)    │
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
┌───────┐ ┌───────┐
│APPROVE│ │REJECT │
│(exec) │ │(loop) │
└───┬───┘ └───┬───┘
    │         │
    ▼         └──────► Vuelve a AI AGENT
┌───────┐
│RESULTS│
└───────┘
```

**Reglas:**
- El nodo "HUMAN REVIEW" siempre lleva el verde
- Debe ser visualmente mas grande que los demas
- El loop de rechazo es visible
- Nunca mostrar AI ejecutando sin supervision humana

---

### 5.3 Diagrama: Learning Loop

Ciclo de aprendizaje continuo. MAXIMO 4 nodos.

```
         ┌───────────┐
         │   LEARN   │
         │ (observe) │
         └─────┬─────┘
               │
     ┌─────────┴─────────┐
     │                   │
     ▼                   │
┌─────────┐              │
│  BUILD  │              │
│(create) │              │
└────┬────┘              │
     │                   │
     ▼                   │
┌──────────┐             │
│ OPTIMIZE │             │
│ (refine) │             │
└────┬─────┘             │
     │                   │
     ▼                   │
┌──────────┐             │
│  SCALE   │ ────────────┘
│ (grow)   │  ◄── Verde: el objetivo final
└──────────┘
```

**Variantes permitidas:**
- Learn → Build → Test → Scale
- Observe → Create → Measure → Improve
- Input → Process → Analyze → Optimize

**Regla:** Siempre 4 nodos. Ni mas, ni menos.

---

### 5.4 Diagrama: Value Ladder

Para mostrar progresion de valor/precio/compromiso.

```
                                    ┌─────────────┐
                                    │ ENTERPRISE  │
                                    │   $$$$$     │
                               ┌────┴─────────────┘
                               │
                          ┌────┴────┐
                          │ PREMIUM │
                          │  $$$$   │
                     ┌────┴─────────┘
                     │
                ┌────┴────┐
                │LOW TIER │
                │   $$    │
           ┌────┴─────────┘
           │
      ┌────┴────┐
      │  FREE   │ ◄── Verde: punto de entrada
      │   $0    │
      └─────────┘
```

**Reglas:**
- Siempre ascendente (izq a der o abajo a arriba)
- 4 niveles maximo
- Verde en el punto de entrada O en el objetivo
- Etiquetas de precio/valor opcionales

## 6. Ilustraciones (Whiteboard Cartoon)

### Sistema de Personajes

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     WHITEBOARD CARTOON STYLE                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  CARACTERISTICAS:                                                       │
│  • Personajes simples, tipo stick figure evolucionado                   │
│  • Proporciones exageradas para claridad                                │
│  • Expresiones minimas pero claras                                      │
│  • Trazo marker/pizarron                                                │
│  • MONOCROMATICO (negro sobre blanco o blanco sobre negro)              │
│                                                                          │
│  LO QUE NO ES:                                                          │
│  • No es cute/kawaii                                                    │
│  • No es cartoon colorido                                               │
│  • No es ilustracion editorial compleja                                 │
│  • No es 3D o semi-realista                                             │
│                                                                          │
│  REFERENCIA MENTAL:                                                     │
│  "Dibujado en 30 segundos en un pizarron durante una reunion"           │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### Arquetipos de Personaje

| Arquetipo | Uso | Elementos Visuales |
|-----------|-----|-------------------|
| El Estratega | Explicando sistemas | Pizarron, diagrama al fondo |
| El Confundido | Antes del framework | Signos de interrogacion |
| El Iluminado | Despues del framework | Foco/bombilla simple |
| El Ejecutor | Implementando | Laptop, herramientas |

### Reglas de Ilustracion

1. **Un personaje = una idea**
2. **Contexto minimo** (solo lo necesario para entender)
3. **Sin fondos elaborados**
4. **Verde solo si el personaje es el punto clave**

## 9. Redes Sociales Sistema

### Formatos por Plataforma

| Plataforma | Formato Principal | Estilo Dominante |
|------------|-------------------|------------------|
| LinkedIn | Carousel educativo | Whiteboard 80% |
| Instagram | Reels + Carousels | Hibrido 50/50 |
| YouTube | Thumbnails + Videos | Executive UI 60% |
| Twitter/X | Quote cards | Minimalista |

### Templates Consistentes

**Carousel Educativo:**
- Slide 1: Hook (pregunta o statement bold)
- Slides 2-6: Framework/Diagrama
- Slide final: CTA + @handle

**Quote Card:**
- Fondo blanco o negro solido
- Cita centrada
- Subrayado verde en palabra clave
- Sin decoracion extra

**Thumbnail:**
- Texto bold (3-5 palabras max)
- Un elemento whiteboard
- Alto contraste
- Verde en un punto focal

## 11. Test Final de Calidad

### Las 5 Preguntas (Obligatorias)

Antes de aprobar CUALQUIER grafico, responde:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      QUALITY TEST                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  1. ¿Esto explica un sistema?                                           │
│     → Si es solo decorativo, eliminalo                                  │
│                                                                          │
│  2. ¿Se entiende sin contexto?                                          │
│     → Si necesita explicacion verbal, simplificalo                      │
│                                                                          │
│  3. ¿Hay un humano dirigiendo?                                          │
│     → Si la AI parece autonoma, agrega el human loop                    │
│                                                                          │
│  4. ¿El verde esta senalando, no decorando?                             │
│     → Si el verde no apunta a algo critico, quitalo                     │
│                                                                          │
│  5. ¿Se siente calmado, no ansioso?                                     │
│     → Si genera urgencia artificial, suavizalo                          │
│                                                                          │
│  REGLA: Si falla UNA pregunta, no se aprueba.                           │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```
