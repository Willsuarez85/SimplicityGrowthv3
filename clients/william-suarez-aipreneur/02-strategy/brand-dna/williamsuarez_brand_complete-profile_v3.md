## 5. Brand Voice

### Voice Characteristics

| Attribute | Description | Example |
|-----------|-------------|---------|
| **Clear** | No jargon without explanation | "MCP es como el USB-C - conecta todo" |
| **Direct** | Gets to the point | "Esto es lo que funciona. Esto no." |
| **Confident** | Speaks from experience | "Asi lo hago en mi agencia" |
| **Practical** | Always actionable | "El paso 1 es..." |
| **Honest** | Includes limitations | "Esto NO funciona para X" |

### Tone Spectrum

```
Formal ─────────────────────────────── Casual
                    ▲
              [AIPreneur]
         Professional but accessible
```

### Language Guidelines

**DO:**
- Use Spanish naturally (not translated-feeling)
- Include technical terms WITH explanations
- Reference real Simplicity examples
- Speak as peer, not guru

**DON'T:**
- Use buzzwords without substance
- Promise without process
- Overcomplicate explanations
- Use English unnecessarily

### The Voice Test

Every piece of content should sound like:
> "Un amigo que sabe de sistemas explicandote algo complejo de forma simple mientras toman cafe."

## 7. The 4 Content Pillars

### Pillar Distribution

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                          │
│  PILAR 1           PILAR 2          PILAR 3           PILAR 4           │
│  IA Aplicada  →    Sistemas    →    Metodo      →     Marco Mental      │
│  a Operaciones     sin Codigo       de Implementacion  del Operador     │
│                                                                          │
│  "Donde usar"      "Como disenar"   "Paso a paso"     "Como pensar"     │
│                                                                          │
│  30%               30%              25%                15%               │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### Pillar 1: IA Aplicada a Operaciones (30%)

**Message:** "La IA sirve para que tu negocio funcione mejor, no para verse cool."

**Purpose:** Educate WHERE to apply AI in business

**Content Types:**
- Where to automate first (quick wins)
- ROI analysis of automation
- Common mistakes to avoid
- Real Simplicity case studies

**Emotional Shift:**
- FROM: "No se donde empezar"
- TO: "Ya se exactamente que proceso automatizar primero"

### Pillar 2: Sistemas sin Codigo (30%)

**Message:** "No necesitas saber codigo, necesitas saber disenar."

**Purpose:** Teach HOW to build systems without programming

**Content Types:**
- Architecture and system design
- No-code tools (Make, n8n, Claude)
- Build-alongs step by step
- Claude Code for non-programmers

**Emotional Shift:**
- FROM: "Esto es para programadores"
- TO: "Puedo disenar sistemas sin escribir codigo"

**Core Tools:**
1. Claude Code - Primary agent tool
2. n8n / Make - Flow automation
3. Notion / Airtable - No-code databases
4. MCP - Connection protocol

### Pillar 3: Metodo de Implementacion (25%)

**Message:** "La diferencia no es la herramienta, es el metodo."

**Purpose:** Provide replicable implementation frameworks

**Content Types:**
- Step-by-step processes
- Mini-frameworks and checklists
- Common errors and solutions
- Templates and resources

**Emotional Shift:**
- FROM: "Intente automatizar pero siempre se rompe"
- TO: "Tengo un metodo que funciona cada vez"

**Core Framework: IDEAL**
1. **I**dentify - Identify the process
2. **D**esign - Design the solution
3. **E**xecute - Build the system
4. **A**nalyze - Measure results
5. **L**everage - Scale what works

### Pillar 4: Marco Mental del Operador (15%)

**Message:** "El futuro favorece a quien ejecuta mejor, no a quien sabe mas."

**Purpose:** Build the AIPreneur mindset

**Content Types:**
- Mindset and identity content
- Strategic decision-making
- Why some implement faster
- Long-term vision

**Emotional Shift:**
- FROM: "Estoy atras y no se como alcanzar"
- TO: "Pienso como operador, no como usuario"

### Mode Selection

| Mode | Background | Stroke | Use Case |
|------|------------|--------|----------|
| Light | #EDEDED (paper gray) | Black (#000000) | Tutorials, explainers |
| Dark | #2B2B2B (executive dark) | White (#FFFFFF) | Impact, authority |

**Rule:** Never mix modes in a single asset.

### Visual Elements

**Allowed:**
- Simple boxes and rectangles
- Circles and ellipses
- Arrows (straight and curved)
- Underlines and highlights
- Check marks (green #87A96B for emphasis)
- Simple line-drawn icons
- Flow connections
- Whiteboard cartoon characters (monochromatic)

**Not Allowed:**
- Decorative illustrations
- Complex graphics
- Photorealistic elements in diagrams
- 3D effects or renders
- Shadows and gradients
- Colored fills (except single green accent)
- Patterns or textures
- Neon colors
- Emojis
- Futuristic/tech hype visuals
- Influencer aesthetics

### 5-Question Quality Test

Before approving ANY visual:

| # | Question | Required Answer |
|---|----------|-----------------|
| 1 | Does this explain a system? | YES |
| 2 | Is it understandable without context? | YES |
| 3 | Is there a human directing? | YES (visible) |
| 4 | Is green signaling, not decorating? | YES |
| 5 | Does it feel calm, not anxious? | YES |

**If any answer is NO, revise before publishing.**

```

□ High contrast maintained throughout
□ Typography is geometric sans-serif
□ Diagrams follow whiteboard/marker style
□ No decorative or unnecessary elements
□ Clear visual hierarchy established
□ Consistent with: "Clarity over personality"
```
