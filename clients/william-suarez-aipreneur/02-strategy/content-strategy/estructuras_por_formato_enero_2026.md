## FORMATO 1: REEL 30 SEGUNDOS

### Objetivo
Quick win educativo o insight rápido que genera "save" o "share"

### Timing Exacto
- **0-3 seg:** Hook visual + textual
- **3-12 seg:** Setup del problema/contexto
- **12-25 seg:** Solución/insight core
- **25-30 seg:** Payoff + CTA suave

### Estructura Narrativa

```
[APERTURA - 3 seg]
→ Hook que detiene scroll
→ NO saludo, directo al punto

[PROBLEMA/CONTEXTO - 9 seg]
→ Establece situación reconocible
→ "Si haces X, probablemente Y"

[SOLUCIÓN/INSIGHT - 13 seg]
→ Resolución concreta
→ Máximo 3 pasos o 1 insight fuerte

[CIERRE - 5 seg]
→ Recapitulación de 1 frase
→ CTA: "Guarda esto" o pregunta engagement
```

### Reglas Visuales
- **Texto en pantalla:** Refuerza audio, no reemplaza
- **Cortes:** Cada 2-3 segundos (mantiene atención)
- **Transiciones:** Cuts duros, no efectos elaborados
- **Fondo:** Face to camera O screen recording, nunca stock footage

### Ejemplo: ENE-002
```
[0-3s] "Si haces esto más de 3 veces al mes..."
       → Pantalla: "3 VECES = PUNTO DE QUIEBRE"

[3-12s] "...probablemente estás usando herramienta cuando necesitas agente."
        → Visual: Persona repitiendo tarea manualmente

[12-25s] "Herramienta: tú la usas. Agente: trabaja por ti.
          Si es repetible, delegable, y tiene criterio → agente."
          → Visual: Comparación lado a lado

[25-30s] "¿Qué tarea haces más de 3 veces al mes?"
         → Pantalla: "COMENTA ABAJO"
```

### CTAs Efectivos (30s)
- "Guarda esto para [situación]"
- "¿Te pasa esto?"
- "¿Cuál de estos usas tú?"
- "Comenta [pregunta específica]"

### KPIs Objetivo
- **Saves:** >8% (contenido útil guardable)
- **Shares:** >3% (contenido compartible)
- **Completion rate:** >60%
- **Comments:** >50 por video

## FORMATO 2: REEL 60 SEGUNDOS

### Objetivo
Mostrar proceso, método o case study con profundidad suficiente para entender

### Timing Exacto
- **0-3 seg:** Hook visual + textual fuerte
- **3-15 seg:** Setup problema o contexto
- **15-50 seg:** Desarrollo (pasos, proceso, insights)
- **50-60 seg:** Payoff + CTA estratégico

### Estructura Narrativa

```
[APERTURA - 3 seg]
→ Hook con resultado o contraste dramático
→ Establece tensión inmediata

[SETUP - 12 seg]
→ Problema detallado o situación "antes"
→ Por qué importa esto

[DESARROLLO - 35 seg]
→ Método/proceso/solución en 3-5 pasos
→ Cada paso máximo 7 segundos
→ Visual apoya cada paso

[CIERRE - 10 seg]
→ Resultado/transformación
→ CTA que invita a acción o reflexión
```

### Reglas de Ritmo
- **Cambio visual:** Cada 3-5 segundos
- **Pasos máximos:** 3-5 (más = confusión)
- **Números:** Siempre usar si hay secuencia
- **Música:** Opcional, nunca sobre voz

### Ejemplo: ENE-006
```
[0-3s] "Antes: 2 horas por cliente. Ahora: 8 minutos."
       → Split screen: reloj 2h vs 8min

[3-15s] "Mi onboarding era manual. Email, calendario, docs, seguimiento.
         Todo a mano. Cada. Vez."
         → Montaje rápido de tareas manuales

[15-25s] "Paso 1: Identifiqué lo que siempre se repite"
          → Screen: checklist de pasos recurrentes

[25-35s] "Paso 2: Construí agente que hace intake inicial"
          → Screen: agente respondiendo preguntas

[35-45s] "Paso 3: Human in the loop para aprobación final"
          → Diagrama: AGENTE → HUMANO → CLIENTE

[45-50s] "Resultado: cliente recibe bienvenida en 8 minutos.
          Yo solo apruebo."
          → Visual: notificación de aprobación

[50-60s] "¿Qué proceso de onboarding automatizarías primero?
          Guarda esto si haces onboarding."
          → CTA en pantalla
```

### Variaciones de Estructura

**Variación A: Before/After**
```
[0-15s] Situación ANTES (problema)
[15-45s] Qué cambió (solución/método)
[45-60s] Situación DESPUÉS (resultado)
```

**Variación B: Behind-the-Scenes**
```
[0-10s] "Así funciona mi [sistema] de verdad"
[10-50s] Walkthrough paso a paso
[50-60s] "Esto es lo que cambia cuando lo usas"
```

**Variación C: Myth vs Reality**
```
[0-15s] "La mayoría cree que [mito]"
[15-45s] "En realidad funciona así [realidad]"
[45-60s] "Por eso [consecuencia/acción]"
```

### CTAs Efectivos (60s)
- "Guarda esto y aplícalo esta semana"
- "¿En qué paso estás tú?"
- "Cuéntame en comentarios si te funcionó"
- "Sígueme para la parte 2"

### KPIs Objetivo
- **Saves:** >10% (más valor = más guardados)
- **Watch time:** >75% (contenido completo visto)
- **Profile visits:** >5%
- **Followers from video:** >2%

## FORMATO 3: CARRUSEL (5-8 SLIDES)

### Objetivo
Educación visual estructurada que genera saves y regresa como referencia

### Número Ideal de Slides
- **Mínimo:** 5 (menos = no justifica formato)
- **Óptimo:** 6-7 (balance profundidad/atención)
- **Máximo:** 8 (más = abandono)

### Estructura por Slides

```
SLIDE 1: PORTADA/HOOK
→ Título claro del framework/concepto
→ Subtítulo con promesa específica
→ Visual: limpio, texto grande, sage green accent

SLIDES 2-6: CONTENIDO CORE
→ 1 idea por slide
→ Máximo 15 palabras por slide
→ Diagrama/visual supporting cada punto
→ Numeración clara (Paso 1, Paso 2...)

SLIDE 7-8: CIERRE/CTA
→ Recapitulación visual (roadmap completo)
→ CTA: "Guarda para cuando..." o "Comparte con..."
→ Opcional: siguiente paso o recurso
```

### Reglas de Diseño

**Tipografía:**
- Headline: 32-40pt (legible en móvil)
- Body: 18-24pt
- Sans-serif geométrica (marca AIPreneur)

**Color:**
- Background: Negro (#000000) o Blanco (#FFFFFF)
- Texto: Blanco o Negro (contraste máximo)
- Accent: Sage Green (#87A96B) solo en 1 elemento por slide
- Grises: Para información secundaria

**Layout:**
- Márgenes: 10% por lado
- Jerarquía clara: título → visual → texto soporte
- Whitespace generoso (no apretujar)

**Visual:**
- Diagramas whiteboard-style
- Iconos minimalistas (opcional)
- NO fotos de stock
- NO gradientes o 3D

### Ejemplo: ENE-005 Framework IDEAL

```
SLIDE 1: PORTADA
"Framework IDEAL"
"Del caos a sistema en 5 pasos"
[Visual: Logo/símbolo del framework]

SLIDE 2: IDENTIFY
"1. IDENTIFICA qué se repite"
"Si lo haces 3+ veces, es candidato"
[Visual: Loop de tarea repetitiva]

SLIDE 3: DESIGN
"2. DISEÑA el flujo ideal"
"Papel primero, código después"
[Visual: Sketch de flujo con cajas y flechas]

SLIDE 4: EXECUTE
"3. EJECUTA con agente"
"Construye, no configures"
[Visual: Input → Agente → Output]

SLIDE 5: ANALYZE
"4. ANALIZA qué falla"
"El primer intento nunca es perfecto"
[Visual: Checklist con algunos X]

SLIDE 6: LEVERAGE
"5. LEVERAGEA lo que funciona"
"Sistema > herramienta aislada"
[Visual: 1 pieza → sistema completo]

SLIDE 7: ROADMAP
"IDEAL en acción"
[Visual: Los 5 pasos conectados en diagrama]

SLIDE 8: CTA
"Guarda esto para tu próxima automatización"
"¿En qué paso estás ahora?"
```

### Ejemplo: ENE-011 Anatomía de un Agente

```
SLIDE 1: "Las 5 piezas que todo agente necesita"

SLIDE 2: "1. INPUT - Cómo recibe información"
SLIDE 3: "2. CRITERIO - Cómo decide qué hacer"
SLIDE 4: "3. EJECUCIÓN - Cómo actúa"
SLIDE 5: "4. VALIDACIÓN - Cómo verifica calidad"
SLIDE 6: "5. OUTPUT - Cómo entrega resultado"

SLIDE 7: "Estas 5 piezas conectadas = agente funcional"
SLIDE 8: "¿Cuál pieza te falta en tu agente?"
```

### CTAs Efectivos (Carrusel)
- "Guarda esto para [situación específica]"
- "Comparte con quien necesita esto"
- "¿En qué paso/pieza estás?"
- "Descarga la versión completa en [link bio]"

### KPIs Objetivo
- **Saves:** >15% (formato más "guardable")
- **Carousel cards seen:** >60% de slides vistos
- **Shares:** >4%
- **Profile visits:** >6%

## FORMATO 4: YOUTUBE LONG-FORM (10-20 MIN)

### Objetivo
Establecer autoridad mediante tutorial profundo o case study detallado

### Duración Óptima por Tipo
- **Tutorial básico:** 10-12 min
- **Deep dive técnico:** 15-18 min
- **Case study completo:** 18-25 min
- **Máximo recomendado:** 25 min (atención cae después)

### Estructura Narrativa Completa

```
[HOOK - 0:00-0:15]
→ Resultado o promesa específica
→ "Al final de este video podrás [resultado concreto]"
→ Visual: thumbnail concept en movimiento

[SETUP PROBLEMA - 0:15-1:30]
→ Por qué importa esto
→ Quién es esto para (y para quién NO)
→ Qué vas a aprender exactamente

[CONTEXT/BACKGROUND - 1:30-3:00]
→ Conceptos necesarios para entender
→ Herramientas que necesitarás
→ Expectativas realistas

[DESARROLLO PASO A PASO - 3:00-17:00]
→ 3-5 secciones principales
→ Cada sección 3-4 minutos
→ Screen recording + voiceover explicativo
→ Timestamps visibles en video

[TROUBLESHOOTING - 17:00-19:00]
→ Errores comunes y cómo arreglarlos
→ "Si te pasa X, haz Y"
→ Validación que funcionó

[RECAP + NEXT STEPS - 19:00-20:00]
→ Recapitulación rápida de lo logrado
→ Qué hacer después de esto
→ CTA: suscribir, comentar, recurso adicional
```

### Ejemplo: ENE-003 "Tu primer agente con Claude Code"

```
[0:00-0:15] HOOK
"En 12 minutos vas a construir tu primer agente con Claude Code.
 Sin saber programar. Sin editar nada. Empecemos."
→ Visual: pantalla de Claude Code lista

[0:15-1:30] SETUP
"Esto es para ti si:
 - Sabes lo que quieres automatizar
 - No sabes programar (o no quieres)
 - Quieres ver el proceso real, sin trucos

 No vamos a editar nada. Lo que funcione, funciona.
 Lo que falle, lo arreglamos en vivo."

[1:30-3:00] CONTEXT
"Vas a necesitar:
 - Cuenta en Claude (link abajo)
 - Un proceso simple para empezar
 - 12 minutos sin interrupciones

 El objetivo: agente que responde emails de contacto automáticamente."

[3:00-5:00] SECCIÓN 1: Setup inicial
→ Crear proyecto en Claude
→ Configurar contexto del agente
→ Definir qué debe hacer

[5:00-9:00] SECCIÓN 2: Construcción del agente
→ Escribir instrucciones claras
→ Conectar con email (MCP)
→ Probar primera respuesta

[9:00-11:00] SECCIÓN 3: Ajustes y validación
→ Revisar calidad de respuestas
→ Ajustar criterio de decisión
→ Confirmar funcionamiento

[11:00-11:45] TROUBLESHOOTING
"Si el agente no responde:
 - Verifica la conexión MCP
 - Revisa permisos de email
 - Checa que las instrucciones sean claras"

[11:45-12:00] RECAP
"Construiste un agente funcional en 12 minutos.
 Siguiente paso: automatiza otro proceso usando este método.

 Si funcionó, déjame saber en comentarios.
 Suscríbete para más tutoriales sin editar."
```

### Formato Visual

**Screen Recording:**
- 1920x1080 mínimo
- Cursor destacado (círculo amarillo en grabación)
- Zoom in en áreas importantes
- NO música de fondo, solo voz

**Face to Camera:**
- Segmentos de 15-30 segundos intercalados
- Para explicar conceptos o transiciones
- Fondo neutro (marca AIPreneur: oscuro/minimalista)
- Luz frontal suave

**B-Roll/Supporting:**
- Diagramas whiteboard-style
- Texto en pantalla con puntos clave
- Timestamps visibles en secciones

### Timestamps en Descripción

```
0:00 Intro - Qué vamos a construir
0:15 Para quién es esto
1:30 Herramientas necesarias
3:00 Setup inicial
5:00 Construcción del agente
9:00 Ajustes y validación
11:00 Troubleshooting común
11:45 Recap y próximos pasos
```

### CTAs Efectivos (YouTube)

**Soft CTAs (preferidos):**
- "Si esto te funcionó, déjame saber en comentarios"
- "Suscríbete para más tutoriales de Claude Code en español"
- "Link a recursos en la descripción"

**Engagement CTAs:**
- "¿Qué proceso vas a automatizar con esto?"
- "Comenta si te trabaste en algún paso"
- "¿Quieres que haga tutorial de [tema]?"

**Conversion CTAs (esporádicos):**
- "Si quieres ayuda 1-on-1, link en descripción"
- "Únete a la comunidad [enlace]"

### KPIs Objetivo
- **AVD (Average View Duration):** >45%
- **CTR (Click-Through Rate):** >8%
- **Likes:** >4% de vistas
- **Comments:** >30 por video
- **Subscribers from video:** >1% de vistas

## CHECKLIST PRE-PUBLICACIÓN

### Para TODOS los Formatos

**Voz y Tono:**
- [ ] Clara: Una idea por oración
- [ ] Directa: Sin rodeos, al punto
- [ ] Tranquila: Confianza sin arrogancia
- [ ] Didáctica: Explica para entender
- [ ] Sin exageraciones: Cero hype

**Contenido:**
- [ ] Pasa filtro: "¿Ayuda a implementar IA real?"
- [ ] Es accionable (tiene pasos claros)
- [ ] Tiene resultado tangible
- [ ] Alineado con pillar correspondiente
- [ ] Evita palabras bannedas (revolucionario, increíble, etc.)

**Visual (si aplica):**
- [ ] 90-95% monocromático (blanco/negro/grises)
- [ ] 5-8% sage green solo en highlights
- [ ] Tipografía sans-serif geométrica
- [ ] Diagramas estilo whiteboard/marker
- [ ] Sin gradientes, 3D, o futurismo cliché

**Engagement:**
- [ ] Hook funciona standalone (sin contexto previo)
- [ ] CTA claro y específico
- [ ] Genera al menos 1 de: save, share, comment
- [ ] Thumbnail (si YouTube) max 4-5 palabras

## HERRAMIENTAS DE PRODUCCIÓN

- **Screen:** OBS Studio o QuickTime (Mac)

- **Reels/Shorts:** CapCut (templates disponibles)
- **YouTube:** DaVinci Resolve o Premiere Pro

### Assets
- **Diagramas:** Excalidraw (whiteboard style)
